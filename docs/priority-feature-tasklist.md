# 優先機能の実装タスク表

この文書は、現在の完成度を崩しにくい順で、次に手を付ける機能を実装単位へ分解したものです。

対象は次の3つです。

- `4. エラー表示と空状態`
- `1. Undo/Redo`
- `3. 出力の完成`

## 進め方の前提

- 既存の Designer / Layout の責務分離を壊さない
- 大きなデータ構造変更は後回しにする
- まずは失敗しにくい改善から入る
- 途中で戻せる小さな変更単位で進める

## 優先順

### 1. エラー表示と空状態

最初にこれを入れる。

理由:

- 全機能の土台になる
- 実装が比較的小さい
- 失敗時の挙動を先に整えると、後続の機能追加が安全になる

### 2. Undo/Redo

次にこれを入れる。

理由:

- 編集体験の価値が大きい
- ただしコマンド設計が要るので、エラー表示より重い

### 3. 出力の完成

最後にこれを固める。

理由:

- 既存の Layout と印刷系の整合が重要
- UI とデータの両方をまたぐので、土台が整ってからの方が安全

## 1. エラー表示と空状態

### 目的

- 読み込み失敗時に何が起きたか分かるようにする
- 空のプロジェクトでも操作に迷わないようにする
- 用紙に収まらない、画像が読めない、設定が未完了の時の案内を整える

### タスク

#### 1-1. 読み込み失敗メッセージを統一する

- 画像読み込み失敗
- ドキュメント読込失敗
- レイアウトに置けない場合

それぞれの文言を揃える。

#### 1-2. 空状態の表示を整える

- バッジが 0 件のときの案内
- Layout が空のときの案内
- 次に何をすればよいかの短い説明

#### 1-3. 用紙に収まらないときの説明を出す

- 何件が収まらなかったか
- どのサイズが原因か
- どうすれば改善するか

#### 1-4. 読み込み失敗時にクラッシュしない確認

- 壊れた PNG
- 未対応形式
- 空ファイル

### 触る候補

- `imageprocessor.cpp`
- `mainwindow.cpp`
- `layoutworkspacewidget.cpp`
- `designerwidget.cpp`
- `exportdialog.cpp`

### 完了条件

- 失敗時に理由が見える
- 空の画面でも案内がある
- エラー時にアプリが不安定にならない

## 2. Undo/Redo

### 目的

- 編集ミスをすぐ戻せるようにする
- 長時間作業でも安心して触れるようにする

### タスク

#### 2-1. コマンド対象を決める

最初の対象候補:

- 移動
- サイズ変更
- 削除
- 画像差し替え

#### 2-2. コマンドの単位を小さくする

- 1 操作 1 コマンドを基本にする
- ドラッグ中はまとめる
- 連続入力は必要なら1つにまとめる

#### 2-3. 編集状態のスナップショットを整理する

- 元の値
- 変更後の値
- 選択状態の復元方法

#### 2-4. MainWindow から直接いじる箇所を減らす

- 編集命令をコマンドへ寄せる
- UI 更新とデータ更新を分ける

#### 2-5. Undo/Redo UI を足す

- メニュー
- ショートカット
- 必要ならツールバー

### 触る候補

- `mainwindow.cpp`
- `mainwindow.h`
- `designerwidget.cpp`
- `designerwidget.h`
- `badgegraphicitem.cpp`
- `badgegraphicitem.h`
- `layoutworkspacewidget.cpp`

### 完了条件

- 主要編集操作が戻せる
- 連続操作でも履歴が破綻しない
- 選択状態が大きく崩れない

## 3. 出力の完成

### 目的

- 画面の Layout と出力結果の差を減らす
- 印刷と納品で安心して使える状態にする

### タスク

#### 3-1. 出力経路を一本化する

- PNG 出力
- PDF 出力
- 印刷プレビュー

これらで同じ描画経路を使う。

#### 3-2. Layout と出力の一致を確認する

- 余白
- カットライン
- 安全圏
- 解像度

#### 3-3. ビットマップ出力の解像度を固定する

- 既定値を明確にする
- 必要なら 300 DPI ベースに寄せる

#### 3-4. エクスポート失敗時の案内を整える

- 書き込み失敗
- 権限不足
- 出力先が使えない

#### 3-5. 印刷前チェックを追加する

- サイズ確認
- ページ数確認
- 余白確認

### 触る候補

- `exportdialog.cpp`
- `layoutworkspacewidget.cpp`
- `mainwindow.cpp`
- `transferdebugdialog.cpp`
- `SPEC.md`

### 完了条件

- Layout と出力の見え方が大きくズレない
- PNG/PDF の出力が安定する
- 失敗時のメッセージが分かりやすい

## 推奨の着手順

1. 空状態とエラー表示を整える
2. Undo/Redo の対象操作を絞る
3. まず 1 操作 1 コマンドで実装する
4. PNG/PDF 出力経路を確認する
5. 最後に印刷周りの細部を詰める

## 途中で確認したいこと

- Undo/Redo を Designer だけに先行導入するか、Layout まで広げるか
- 出力の完成を「PNG/PDF の安定化」までにするか、「印刷の細部」まで含めるか
- 空状態の案内をステータス表示にするか、ダイアログにするか

## ひとこと

この3つは、派手さはないけれど完成度を支える部分です。  
先に土台を固めると、その後の機能追加がかなり楽になります。
