# 同一デザインのサイズ違いを同時に紙面レイアウトする手順書

この文書は、同じデザインを `32mm` と `57mm` など複数サイズで同数並べたいときの、最小変更で壊しにくい実装手順をまとめたものです。

## 目的

- 既存のデータ構造や保存形式をできるだけ変えない
- Designer の編集体験を壊さない
- Layout 側でだけ、同じデザインのサイズ違いを同時配置できるようにする
- 将来の実装に向けた手順だけ先に固定する

## 前提

- 既存の `BadgeItem` はそのまま使う
- 既存の `BadgeData` / `DocumentData` を大きく拡張しない
- サイズ違いは「別の `BadgeItem`」として扱う
- 画像やレイヤーは再利用し、変えるのは主に `widthMm` / `heightMm`

## 基本方針

### 方針 A

同じデザインを複製し、サイズだけ変えた複数の `BadgeItem` を作る。

例:

- デザインA 32mm x 20枚
- デザインA 57mm x 20枚

### 方針 B

Layout 直前でだけ複製する。

- Designer では 1 つの元デザインを扱う
- Layout へ渡す前に、サイズ別の一時リストへ変換する
- 既存の編集フローはそのまま維持する

## 推奨する実装順

### 1. 複製用の小さな関数を追加する

既存の `BadgeItem` を受け取り、サイズだけ差し替えた新しい `BadgeItem` を返す関数を作る。

必要な変更だけに絞る。

- `widthMm`
- `heightMm`
- 必要なら `label`
- 必要なら `xMm` / `yMm` は Layout 側で再配置するので初期値のままでよい

### 2. サイズ別の入力リストを作る

Layout へ渡す直前に、次のような構造へ変換する。

- 元デザインA 32mm 20枚
- 元デザインA 57mm 20枚

この段階では「同じ数」を保証するだけでよい。

### 3. サイズごとに別レイアウトする

`LayoutEngine` の既存処理を壊さず、サイズごとにまとめて並べる。

候補:

- 32mm のグループを先に配置
- 57mm のグループを後に配置
- あるいは用紙を分ける

最初は「同一紙面に混在」よりも「サイズごとにブロック配置」の方が安全。

### 4. 必要なら紙面分割を追加する

1 枚に収まらないときは、サイズ別にページを分ける。

これなら:

- 32mm が多すぎても崩れにくい
- 57mm が多すぎても扱いやすい
- 既存の自動配置ロジックを流用しやすい

## データの持ち方

最小変更でやるなら、次のどちらかにする。

### 案 1: UI 側で複製する

- 画面上で「サイズ違いを作成」ボタンを押す
- その時点で `BadgeItem` を複製する
- 以後は通常のバッジとして扱う

### 案 2: レイアウト時に複製する

- 元デザインだけ保持する
- レイアウト時にだけサイズ違いを生成する
- 既存の保存データは一切変えない

今のコードベースを壊しにくいのは案 2。

## 実装メモ

- `badge.qtbridge.ixx` は `BadgeItem` と `BadgeData` の変換の中心なので、複製処理を置くならここか、その近くがよい
- `layoutengine.cpp` は「配置」だけに寄せる
- Designer の表示や編集ロジックには手を入れすぎない
- 保存フォーマットの変更は最後まで避ける

## 触る候補のファイル

- `layoutengine.cpp`
- `layoutengine.h`
- `projectsync.cpp`
- `projectsync.h`
- `badge.qtbridge.ixx`
- `mainwindow.cpp`
- 必要なら `docs/` 配下の補助資料

## 低リスクの実装順

### 1. データ複製の補助関数

- 既存の `BadgeItem` をコピーする
- `widthMm` / `heightMm` だけ差し替える
- ラベルに必要なら `32mm` / `57mm` を付ける

### 2. 複製後の配置対象を作る

- 元デザイン1件から、サイズごとの複製群を作る
- 各サイズで同数を持たせる
- この時点では紙面に置かない

### 3. サイズ別の配置

- 既存の `LayoutEngine::autoLayoutGrid` か `fillPageGrid` に渡す
- サイズごとにブロック分けする
- 混在配置は後回しにする

### 4. UI を足すなら最後

- まずは内部処理だけ
- その後、必要なら「同数で複製」ボタンを足す
- 保存形式は変えずに済ませる

## 具体例

たとえば元デザイン `A` を 20 枚ずつ作るなら、内部的には次の並びにする。

- `A / 32mm / 20`
- `A / 57mm / 20`

その後に Layout 側で:

- 32mm 群を配置
- 57mm 群を配置
- 必要なら別ページに送る

## 実装時の注意

- サイズ違いを「別デザイン」と誤認しない
- 既存の `label` や `imagePath` を壊さない
- `Designer` の選択状態や表示状態は複製しない方が安全
- `Layout` では `xMm` / `yMm` を再計算するので、複製時の座標は深く気にしなくてよい

## もし将来広げるなら

- 同一デザインの「比率違い」
- 同一デザインの「ミラー版」
- 同一デザインの「色違い」

ただし、まずはサイズ違いだけに絞るのが安全。

## 検証項目

- 32mm と 57mm が同じ枚数で作れる
- 既存の単一サイズの動作が変わらない
- 印刷プレビューで崩れない
- 保存・再読込でサイズ情報が維持される
- 既存のレイアウト自動配置が壊れない

## 失敗しにくい順番

1. 複製関数だけ追加する
2. サイズ違いの一時配列を作る
3. 既存の `LayoutEngine` にその配列を渡す
4. UI は最後に足す

## ひとこと

この機能は、最初から「複数サイズ対応の新モデル」を作るより、既存のバッジを複製して並べる方が安全です。  
今の完成度を崩したくないなら、まずはこの方針で進めるのがよいです。
