# BadgeEditorQt Specification

この文書は、BadgeEditorQt におけるサイズ、DPI、レイアウト、出力の扱いをまとめた仕様書です。

## 1. 基本方針

- 編集データの基準単位は `mm` とする
- 画面表示のための換算は UI 側でのみ行う
- 印刷・書き出し時の DPI は表示用 DPI と分けて扱う
- Designer は作成、Layout は配置確認、Export は出力に責務を分ける

## 2. 単位と座標

- `BadgeItem.widthMm` / `heightMm` / `xMm` / `yMm` はすべて mm 基準
- `PaperConfig.widthMm` / `heightMm` / `marginMm` / `spacingMm` も mm 基準
- 画面上のピクセル換算は `96 DPI` を仮定した表示用変換を使う
- 画面上の 1 mm は `96 / 25.4 px` として扱う

## 3. 既定値

### 用紙

- 既定の用紙サイズは `A4` とする
- `A4` は `210 x 297 mm`
- 主要な用紙サイズとして `A3` / `A4` / `A5` / `B4` / `B5` / `Letter` / `Legal` を扱う
- `A3` は `297 x 420 mm`
- `A5` は `148 x 210 mm`
- `B4` は `257 x 364 mm`
- `B5` は `182 x 257 mm`
- `Letter` は `216 x 279 mm`
- `Legal` は `216 x 356 mm`
- 既定の余白は `10 mm`
- 既定の間隔は `1 mm`

### バッジサイズの代表プリセット

- `25 mm`
- `32 mm`
- `44 mm`
- `57 mm`
- `65 mm`
- `76 mm`
- `100 mm`

## 4. Designer の仕様

- Designer は 1 個のバッジテンプレートを作る場所とする
- 画像、文字、レイヤー、色補正、ガイドを編集する
- Designer 内の編集結果は Layout に自動で大量反映しない
- バッジの見た目は実寸ベースで保持する
- 画面上の初期ズームは操作しやすい倍率を使ってよい

## 5. Layout の仕様

- Layout は用紙上にバッジを並べる場所とする
- Layout は Designer の編集対象とは別のレイヤーとして扱う
- Layout の表示は実寸準拠を優先する
- Layout には用紙枠、安全圏、カットラインを表示してよい
- Layout に送るのは「配置用のバッジ群」であり、Designer の編集状態そのものではない

## 6. レイアウト配置

- バッジの配置計算は mm 基準で行う
- 間隔は `PaperConfig.spacingMm` を使う
- 余白は `PaperConfig.marginMm` を使う
- 自動配置は 1 枚の用紙に収まる範囲で行う
- 1 枚に収まらない場合は、収まる分だけ配置してよい

## 7. 画像読み込み

- ラスター画像は OIIO を主軸に読む
- Qt の画像読み込みは補助扱いにする
- PNG / JPEG / WebP / AVIF など、OIIO が対応する形式は OIIO を優先する
- 読み込み後は表示用 sRGB に正規化する
- 埋め込み色空間を読める場合は、表示ラベルに反映する

## 8. ベクター画像

- ベクター画像は将来対応対象とする
- まずは SVG を第一候補として扱う
- ベクターはラスターと同じ Badge として見えるが、内部では別経路で描画してよい
- 将来的に別レンダラを導入する余地を残す

## 9. 出力

- 出力の基準は mm とする
- ビットマップ出力の DPI は出力時に決める
- 画面プレビューの 96 DPI と、出力 DPI を混同しない
- 既定の出力 DPI は未確定でよいが、印刷用途では 300 DPI 以上を想定する
- 画像出力と PDF 出力は、Layout の見え方にできるだけ一致させる

## 10. 印刷ダイアログ

- 標準の印刷ダイアログを初期の基盤とする
- 将来的にはアプリ独自の印刷設定ダイアログを追加してよい
- 独自ダイアログでは、用紙、余白、カットライン、実寸確認を扱う

## 11. 判断ルール

- 迷ったら mm を正とする
- 画面の見た目が必要なときだけ DPI 換算する
- バッジの元データと用紙設定を分けて考える
- Designer と Layout の責務を混ぜない
